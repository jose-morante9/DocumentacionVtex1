Theme Store

Guía completa sobre Store Theme en VTEX IO

Store Theme define la apariencia de tu tienda

Definición: Es la aplicación que controla toda la interfaz visual y estructura de tu tienda VTEX IO.

Componentes principales
  • Bloques JSON: Estructura de páginas y componentes
  • Estilos CSS: Personalización visual
  • Assets: Imágenes, fuentes, iconos
  • Configuraciones: Comportamiento de componentes
Arquitectura:
store-theme/
├── store/
│ ├── blocks/
│ │ ├── home.jsonc
│ │ ├── product.jsonc
│ │ └── search.jsonc
│ └── interfaces.json
├── styles/
│ ├── css/
│ └── configs/
├── assets/
└── manifest.json
Concepto clave: Todo se construye con bloques JSON que VTEX renderiza como React.

Pasos para iniciar

1 Clonar template base

Usa el template oficial de VTEX:

vtex init
? Choose where to start: store-theme

O clona directamente:

git clone https://github.com/vtex-apps/store-theme.git
2 Configurar manifest.json

Personaliza tu theme:

{
"vendor": "mitienda",
"name": "store-theme",
"version": "0.0.1",
"builders": {
"styles": "2.x",
"store": "0.x",
"assets": "0.x"
},
"dependencies": {
"vtex.store": "2.x",
"vtex.store-header": "2.x",
"vtex.product-summary": "2.x",
"vtex.flex-layout": "0.x",
"vtex.rich-text": "0.x"
}
}
3 Instalar dependencias
cd store-theme
vtex link

Esto instala todas las apps necesarias

4 Crear workspace de desarrollo
vtex use dev-workspace

Trabajarás en: dev-workspace--tuaccount.myvtex.com

5 Ver tu tienda
vtex browse

Abre automáticamente tu tienda en el navegador

Importante: Siempre trabaja en workspace, nunca en master directamente.

Anatomía de un bloque

Ejemplo: Home Page

Archivo: store/blocks/home.jsonc

{
"store.home": {
"blocks": [
"rich-text#banner",
"flex-layout.row#products",
"newsletter"
]
},
"rich-text#banner": {
"props": {
"text": "# ¡Bienvenido a mi tienda!",
"blockClass": "banner-principal"
}
},
"flex-layout.row#products": {
"children": [
"product-summary.shelf"
]
}
}
Bloques más usados:
  • flex-layout: Diseño responsive (filas y columnas)
  • rich-text: Textos con markdown
  • image: Imágenes optimizadas
  • product-summary: Cards de productos
  • search-result: Resultados de búsqueda
  • slider-layout: Carruseles
Ejemplo: Grid de productos
{
"flex-layout.row#shelf": {
"children": ["list-context.product-list#demo"]
},
"list-context.product-list#demo": {
"blocks": ["product-summary.shelf"],
"children": ["slider-layout#products"]
},
"slider-layout#products": {
"props": {
"itemsPerPage": {
"desktop": 4,
"tablet": 3,
"phone": 2
},
"infinite": true,
"showNavigationArrows": "desktopOnly",
"blockClass": "carousel"
}
}
}
Tip: Usa blockClass para aplicar estilos CSS personalizados.

Sistema de estilos en VTEX

1 Configuración de colores

Archivo: styles/configs/style.json

{
"semanticColors": {
"primary": "#F71963",
"secondary": "#142032",
"success": "#28a745",
"danger": "#dc3545",
"warning": "#ffc107",
"muted": "#727273"
},
"borderRadius": "8px",
"fontFamily": "Segoe UI, sans-serif"
}
2 CSS personalizado

Archivo: styles/css/vtex.flex-layout.css

/* Usando blockClass del JSON */
.flexRowContent--banner-principal {
background: linear-gradient(135deg, #F71963, #142032);
padding: 60px 0;
border-radius: 10px;
}

.flexRow--carousel {
margin: 40px 0;
}

.flexCol--product-card {
transition: transform 0.3s;
}

.flexCol--product-card:hover {
transform: translateY(-10px);
}
3 Overrides de componentes

Archivo: styles/css/vtex.product-summary.css

.productBrand {
font-size: 14px;
color: #727273;
text-transform: uppercase;
}

.priceContainer {
color: #F71963;
font-weight: bold;
font-size: 24px;
}

.listPrice {
text-decoration: line-through;
color: #999;
}
Jerarquía de estilos
  1. style.json: Variables globales
  2. vtex.[app].css: Estilos por app
  3. blockClass: Clases específicas
  4. CSS handles: Clases nativas de VTEX
Nota: Los cambios de CSS requieren refresh del navegador (Ctrl + Shift + R).

Comandos principales

Link (desarrollo)
vtex link

Hot reload automático en tiempo real

Abrir tienda
vtex browse

Abre tu workspace en el navegador

Publicar
vtex publish

Publica una nueva versión

Promover a producción
vtex workspace promote

Lleva tu workspace a master

Ver apps instaladas
vtex list
Reinstalar dependencias
vtex deps update
Importante: Siempre prueba en workspace antes de promover a master.
